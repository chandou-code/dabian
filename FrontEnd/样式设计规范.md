## 4. 代码组织规范

### 4.1 CSS命名约定

#### BEM命名规范
项目采用 BEM (Block, Element, Modifier) 命名规范，格式如下：
```
.block__element--modifier
```

**示例：**
```css
/* 块 (Block) */
.sidebar {
  /* 样式 */
}

/* 元素 (Element) */
.sidebar__header {
  /* 样式 */
}

/* 修饰符 (Modifier) */
.sidebar--collapsed {
  /* 样式 */
}
```

#### 命名原则
- 使用小写字母和连字符 `-` 分隔单词
- 避免使用驼峰命名
- 保持命名语义化，清晰表达组件功能
- 组件内部元素使用双下划线 `__` 连接
- 状态修饰使用双连字符 `--` 连接

### 4.2 样式文件组织结构

#### 全局样式
- `uni.scss`：全局变量定义（颜色、字体、间距等）
- `App.vue`：全局样式重置、通用组件样式、工具类

#### 组件样式
- 每个 Vue 组件内部使用 `<style scoped>` 定义组件专属样式
- 组件样式仅作用于当前组件，避免样式污染
- 复用性强的组件样式可提取到全局样式文件

#### 样式文件结构示例
```
FrontEnd/
├── uni.scss              # 全局变量
├── App.vue               # 全局样式
├── components/
│   └── Sidebar.vue       # 组件样式（scoped）
└── pages/
    ├── user/
    │   ├── dashboard.vue # 页面样式（scoped）
    │   └── search.vue    # 页面样式（scoped）
    └── admin/
        └── dashboard.vue # 页面样式（scoped）
```

### 4.3 复用与扩展机制

#### 样式变量复用
- 所有颜色、字体、间距等基础样式均通过 SCSS 变量定义
- 组件样式中优先使用全局变量，保持设计一致性
- 如需自定义组件样式，可在组件内部定义局部变量

#### 工具类复用
```css
/* 文本对齐 */
.text-center { text-align: center; }

/* 文本颜色 */
.text-primary { color: #2196f3; }
.text-success { color: #4caf50; }
.text-warning { color: #ff9800; }
.text-danger { color: #f44336; }
.text-muted { color: #666; }

/* 弹性布局 */
.flex { display: flex; }
.flex-between { display: flex; justify-content: space-between; align-items: center; }
.flex-center { display: flex; align-items: center; justify-content: center; }
.flex-1 { flex: 1; }

/* 字体样式 */
.font-bold { font-weight: bold; }
.font-lg { font-size: 36rpx; }
.font-md { font-size: 30rpx; }
.font-sm { font-size: 24rpx; }
```

#### 组件样式扩展
- 基础组件样式定义在全局，特定组件可通过覆盖样式进行扩展
- 使用 `:deep()` 伪类穿透 `scoped` 样式限制
- 优先使用 CSS 变量进行主题定制

### 4.4 性能优化

#### 样式加载优化
- 避免使用 `@import` 嵌套导入
- 减少不必要的样式层级
- 优先使用 CSS Grid 和 Flexbox 布局

#### 选择器优化
- 避免使用过于复杂的选择器
- 减少选择器层级，控制在 3 层以内
- 优先使用类选择器，避免使用标签选择器和 ID 选择器

#### 样式体积优化
- 移除未使用的样式
- 合并重复样式
- 使用 CSS 变量减少重复代码

## 5. 最佳实践

### 5.1 设计一致性
- 严格遵循设计系统定义的颜色、字体、间距等规范
- 保持组件样式统一，避免个性化设计导致的视觉混乱
- 新增组件时参考现有组件样式，确保设计语言一致

### 5.2 响应式设计
- 所有页面和组件必须适配移动端和桌面端
- 使用相对单位（rpx、%、rem）替代固定像素
- 优先使用弹性布局和网格布局

### 5.3 可访问性
- 确保文本与背景对比度符合 WCAG 标准
- 为交互元素提供清晰的视觉反馈
- 合理使用 ARIA 属性增强可访问性

### 5.4 性能优化
- 避免使用大量复杂动画
- 优化 CSS 选择器性能
- 合理使用 `scoped` 样式
